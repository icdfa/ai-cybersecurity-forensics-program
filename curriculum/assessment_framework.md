# Assessment Framework: Advanced AI in Cybersecurity and Digital Forensics

## 1. Assessment Philosophy

This program employs a continuous and comprehensive assessment strategy designed to measure both theoretical knowledge and practical skills. The assessment framework is built on the principles of:

-   **Authentic Assessment:** Tasks and projects that reflect real-world challenges in AI security.
-   **Formative Assessment:** Continuous feedback to guide student learning and improvement.
-   **Summative Assessment:** Rigorous evaluation of knowledge and skills at key milestones.

## 2. Assessment Components and Weighting

The final grade for the program will be calculated based on the following components:

| Component | Weight | Description |
| :--- | :--- | :--- |
| **Lab Assignments** | 40% | Weekly hands-on labs to develop practical skills. |
| **Quizzes** | 15% | Bi-weekly quizzes to assess understanding of key concepts. |
| **Quarterly Mini-Projects** | 20% | End-of-quarter projects to apply knowledge from the preceding months. |
| **Midterm Exam** | 10% | A comprehensive exam at the end of Quarter 2. |
| **Final Capstone Project** | 15% | A major project in the final quarter, demonstrating mastery of the program's content. |

## 3. Detailed Breakdown of Assessments

### 3.1. Lab Assignments (40%)

-   **Frequency:** Weekly.
-   **Format:** Hands-on coding exercises, practical tasks, and short reports.
-   **Grading:** Based on correctness, code quality, and the clarity of the report.

### 3.2. Quizzes (15%)

-   **Frequency:** Bi-weekly (every two weeks).
-   **Format:** Multiple-choice, short answer, and code snippet analysis questions.
-   **Objective:** To ensure students are keeping up with the theoretical concepts.

### 3.3. Quarterly Mini-Projects (20%)

-   **Frequency:** At the end of each of the first three quarters.
-   **Format:** A multi-week project that integrates the skills and knowledge from the quarter.
-   **Examples:**
    -   **Q1:** Build a complete malware detection engine.
    -   **Q2:** Develop an automated forensic investigation tool.
    -   **Q3:** Conduct a red team assessment of a live AI system.

### 3.4. Midterm Exam (10%)

-   **Timing:** At the end of Quarter 2 (Month 6).
-   **Format:** A combination of theoretical questions and a practical, hands-on challenge.
-   **Scope:** Covers all material from Quarters 1 and 2.

### 3.5. Final Capstone Project (15%)

-   **Timing:** Quarter 4 (Months 10-12).
-   **Format:** A team-based project where students propose, design, build, and present a novel AI-powered cybersecurity or digital forensics solution.
-   **Components:**
    -   Project Proposal
    -   Design Document
    -   Final Implementation and Codebase
    -   Written Report
    -   Final Presentation and Defense

## 4. Grading Scale

| Grade | Percentage | Description |
| :--- | :--- | :--- |
| **A** | 90-100% | Excellent mastery of all concepts and skills. |
| **B** | 80-89% | Good understanding with minor gaps in knowledge or application. |
| **C** | 70-79% | Satisfactory understanding, but with some clear areas for improvement. |
| **D** | 60-69% | Minimal understanding of the material. |
| **F** | Below 60% | Insufficient understanding to pass the program. |

## 5. Academic Integrity

All work submitted must be the student's own. Any instances of plagiarism or academic dishonesty will be taken seriously and may result in a failing grade for the assignment, the course, or the entire program.
