## Program Overview

This 12-month intensive program combines theoretical knowledge with extensive hands-on practical experience, covering both defensive and offensive AI techniques in cybersecurity and digital forensics. The program is designed for cybersecurity professionals, digital forensics investigators, incident responders, and IT professionals transitioning into security roles.

The program addresses a critical market gap by providing comprehensive training in:

- **Defensive AI:** Building secure AI systems and defending against AI-powered threats
- **Offensive AI:** Understanding adversarial attacks, evasion techniques, and AI-based attack methodologies
- **Digital Forensics:** Leveraging AI to automate and enhance forensic investigations
- **Governance & Ethics:** Understanding AI risk management frameworks and ethical considerations

---

## Program Structure

The program is organized into **4 quarters**, each lasting **3 months** (12 weeks), for a total of **52 weeks** of intensive study.

### Quarter 1: Foundations of AI and Cybersecurity (Months 1-3)

Introduces AI, machine learning, and their applications in cybersecurity. Students learn Python programming for security, fundamentals of machine learning, and how to build AI-powered threat detection systems.

**Topics:**
- Python Programming for Cybersecurity
- Fundamentals of AI and Machine Learning
- Data Collection and Processing for Threat Intelligence
- Machine Learning for Threat Prediction and Forecasting
- Static and Dynamic Malware Analysis with AI
- Advanced Malware Detection Techniques (Deep Learning)

**Mini-Project:** Building an Intelligent Malware Detection System

### Quarter 2: AI-Powered Digital Forensics and Incident Response (Months 4-6)

Focuses on using AI to automate and enhance digital forensic investigations and incident response processes.

**Topics:**
- Principles of Digital Forensics and Evidence Handling
- AI for Automated Evidence Collection and Triage
- AI for Timeline Reconstruction and Anomaly Detection
- Natural Language Processing (NLP) for Forensic Analysis
- Computer Vision in Digital Forensics
- AI-Driven Incident Response and SOAR Platforms

**Mini-Project:** Developing an AI-Powered Forensic Analysis Tool

### Quarter 3: Offensive AI and Adversarial Machine Learning (Months 7-9)

Explores the techniques of adversarial AI and the use of AI in offensive cyber operations. Students learn how attackers use AI and how to defend against these threats.

**Topics:**
- Understanding the Adversarial AI Threat Landscape
- Evasion Attacks and Model Robustness
- Data Poisoning, Model Inversion, and Inference Attacks
- Deepfakes, AI-Generated Malware, and Advanced Phishing
- AI Red Teaming Methodologies and Frameworks
- Continuous AI Security Testing and Validation

**Mini-Project:** Conducting a Red Team Assessment of an AI System

### Quarter 4: AI Governance, Capstone Project, and Career Preparation (Months 10-12)

Addresses governance and ethical aspects of AI in security, completion of a comprehensive capstone project, and career preparation.

**Topics:**
- AI Risk Management Frameworks (NIST AI RMF, EU AI Act)
- Ethical Hacking and Responsible Disclosure for AI Systems
- Capstone Project Development
- Career Workshops and Professional Development

**Final Deliverable:** Capstone Project with presentation and defense

## Lab Assignments

### Quarter 1: Foundations of AI and Cybersecurity

1. **[Lab 1: Automating Network Scanning and Packet Analysis](labs/q1/month1/lab1_network_scanning.md)** (Weeks 1-2)
   - Learn Scapy, host discovery, port scanning, packet analysis
   - Develop practical network reconnaissance skills

2. **[Lab 2: Developing a Threat Intelligence Data Pipeline](labs/q1/month2/lab2_threat_intel_pipeline.md)** (Weeks 5-6)
   - Build system to collect threat intelligence from public APIs
   - Normalize and store data in structured database
   - Query and analyze threat data for security insights

3. **[Lab 3: Building an Intelligent Malware Detection System](labs/q1/month3/lab3_intelligent_malware_detection.md)** (Weeks 9-12)
   - Process large-scale malware datasets
   - Engineer and select relevant features for classification
   - Train and compare multiple machine learning models
   - Deploy functional malware detection tool
   - **Q1 Mini-Project**

### Quarter 2: AI-Powered Digital Forensics and Incident Response

4. **[Lab 4: Forensic Analysis of a Compromised System](labs/q2/month4/lab4_forensic_analysis.md)** (Weeks 13-14)
   - Use Autopsy for disk image analysis
   - Apply Volatility Framework for memory forensics
   - Identify and extract evidence of intrusion
   - Correlate findings to reconstruct attack timeline

5. **[Lab 5: Automated Attack Timeline Generation](labs/q2/month5/lab5_timeline_generation.md)** (Weeks 17-18)
   - Parse and normalize logs from multiple sources
   - Correlate events across different log files
   - Visualize attack paths using graph analysis
   - Automate timeline generation for forensic investigations

6. **[Lab 6: Developing a SOAR Playbook for Automated Response](labs/q2/month6/lab6_soar_playbook.md)** (Weeks 23-24)
   - Design workflows for automated incident response
   - Integrate external security tools and APIs
   - Extract and analyze observables from alerts
   - Simulate SOAR playbook execution
   - **Q2 Mini-Project**

### Quarter 3: Offensive AI and Adversarial Machine Learning

7. **[Lab 7: Research and Presentation on Recent Adversarial Attacks](labs/q3/month7/lab7_adversarial_attack_research.md)** (Weeks 25-26)
   - Conduct in-depth research on real-world adversarial attacks
   - Analyze attack techniques and defensive measures
   - Develop research and presentation skills

8. **[Lab 8: Bypassing a Malware Classifier with an Evasion Attack](labs/q3/month7/lab8_evasion_attack.md)** (Weeks 27-28)
   - Implement the Fast Gradient Sign Method (FGSM)
   - Generate adversarial examples to fool classifiers
   - Evaluate attack effectiveness
   - Understand vulnerabilities in AI-based security

9. **[Lab 9: Executing a Data Poisoning Attack on a Learning Model](labs/q3/month8/lab9_data_poisoning_attack.md)** (Weeks 29-30)
   - Inject mislabeled data into training sets
   - Measure model performance degradation
   - Visualize the impact of data poisoning
   - Understand data integrity importance

10. **[Lab 10: Building and Detecting Deepfake Videos](labs/q3/month8/lab10_deepfake_detection.md)** (Weeks 31-32)
    - Generate deepfake videos using AI techniques
    - Implement detection mechanisms
    - Analyze implications for security and society

11. **[Lab 11: Planning and Executing an AI Red Team Exercise](labs/q3/month9/lab11_ai_red_teaming.md)** (Weeks 33-34)
    - Simulate real-world attacks on AI systems
    - Find vulnerabilities in AI deployments
    - Develop red teaming methodologies

12. **[Lab 12: Building an Automated AI Security Validation Pipeline](labs/q3/month9/lab12_ai_security_validation.md)** (Weeks 35-36)
    - Create automated security testing frameworks
    - Integrate multiple adversarial attacks
    - Generate security validation reports
    - Implement continuous monitoring
    - **Q3 Mini-Project**

## Assessment Methods

The program employs multiple assessment methods to ensure comprehensive evaluation:

| Assessment Type | Weight | Description |
| :--- | :--- | :--- |
| **Lab Assignments** | 40% | Weekly hands-on labs to develop practical skills |
| **Quizzes** | 15% | Bi-weekly quizzes to assess theoretical knowledge |
| **Quarterly Mini-Projects** | 20% | End-of-quarter projects integrating quarter's content |
| **Midterm Exam** | 10% | Comprehensive exam at the end of Quarter 2 |
| **Final Capstone Project** | 15% | Major project demonstrating mastery of program content |

### Grading Scale

| Grade | Percentage | Description |
| :--- | :--- | :--- |
| **A** | 90-100% | Excellent mastery of all concepts and skills |
| **B** | 80-89% | Good understanding with minor gaps in knowledge or application |
| **C** | 70-79% | Satisfactory understanding, but with some clear areas for improvement |
| **D** | 60-69% | Minimal understanding of the material |
| **F** | Below 60% | Insufficient understanding to pass the program |

## Capstone Project

In the final quarter, students work in teams to develop a novel AI-powered solution to a real-world cybersecurity or digital forensics challenge.

### Capstone Deliverables

- Project proposal and design document
- Full implementation with well-documented code
- Comprehensive final report (15-20 pages)
- Final presentation and defense

### Suggested Project Areas

- Adversarial attack and defense mechanisms
- AI-powered threat hunting systems
- Deepfake detection and generation
- Automated vulnerability discovery
- AI-driven deception systems (honeypots)

See [Capstone Project Specifications](capstone-project/capstone_project_specifications.md) for more details.

## Resources

### Datasets

Recommended datasets for labs and projects:

- **EMBER:** Large-scale dataset of PE file features for malware classification
- **Sorel-20M:** Large dataset of malware samples with rich metadata
- **CTU-13:** Dataset of botnet traffic captured in real-world network
- **CIC-IDS-2017:** Comprehensive dataset for intrusion detection
- **Digital Corpora:** Repository of disk images, memory dumps, and forensic artifacts

See [Resources/Datasets](resources/datasets/README.md) for comprehensive list.

- **Machine Learning:** Scikit-learn, TensorFlow, Keras, PyTorch

- **Adversarial AI:** ART (Adversarial Robustness Toolbox)

See [Resources/Tools](resources/tools/README.md) for comprehensive list.
